Design infographic · 2026-05-16 Implementation Agent

From intent-confirmed to merged PR — with the reporter and humans both in the loop

The agent that picks up a reporter-confirmed spec, builds it on a per-spec preview env, iterates with the reporter until they're happy, then hands off to humans for review and DevOps for merge. Sits on the v7 portable runtime — no vendor lock-in.

1Three iteration flows — event-driven, all using the same agent core

One agent, three triggers. Each flow uses the same code path inside agents/agents/implementation/core.py; the trigger only decides which entry point runs.

Flow 1 · Initial

⚙️ First implementation

trigger · label intent-confirmed applied to PR on agent/spec-*
  1. Read spec at branch HEAD
  2. Draft plan → commit to docs/superpowers/plans/
  3. Implement code in steps; commit per step
  4. Run abbreviated Gate 1 in agentlab (build · lint · security · tests)
  5. Retry up to 3× on fail; escalate if still red
  6. Spawn preview-spec-<N> Fly app + DB
  7. Seed with masked snapshot, install fixture modules
  8. Provision reviewer login (one-time password)
  9. Post URL + creds + summary to GitHub issue
  10. Label → awaiting-reviewer; Slack ping
Target: < 30 min spec-merge → URL posted.
Flow 2 · Reporter

🔁 Reporter iteration

trigger · issue_comment.created on linked issue (PR label = awaiting-reviewer)
  1. Classify the comment:
    ✓ approval · ↻ iteration · ⨯ out-of-scope · ⚠ escalate
  2. If approval: label → awaiting-human-review, ping CODEOWNERS
  3. If iteration: in-scope check vs spec.goal + spec.non-goals (LLM judgement, confidence ≥ 0.7)
  4. In-scope → coder applies change → re-run Gate 1 → redeploy preview at same URL
  5. Bump iter counter (label iter-N)
  6. If iter > 5 → escalate (needs-human)
  7. If out-of-scope: polite refusal + offer /file-followup
  8. If escalate: label needs-human; page on-call
Cap: 5 iterations. Escalation guarantees a finite loop.
Flow 3 · Human commit · v5 Q15

🛡️ Human refinement + reporter ping

trigger · push to agent/spec-* by non-agent author
  1. Workflow filters bot identities (implementation-bot@, spec-generator-bot@)
  2. Compute diff summary + flag user-visible files (views, qweb, controllers)
  3. Redeploy preview env at same URL
  4. Post (or edit) ping comment on issue with: SHA, author, message, diff, user-visible tag, URL
  5. Debounce: 1-hour window batches commits into one comment
  6. Label: human-review-approved → awaiting-reporter-reconfirm
  7. Start 24h auto-reconfirm timer (resets on each new human commit)
  8. If reporter /approves → awaiting-devops
  9. If silent 24h → auto-clear to awaiting-devops
Cap: 3 human iterations. Branch protection refuses force-push.

7Failure modes & recovery — 13 designed-for failure paths

Agent can't implement

Gate 1 fails 3× in a row.
Label needs-human; preview stays up; humans rewrite spec or implement.

Spec is internally contradictory

Planner self-detects mid-plan.
Label spec-refinement-needed; Spec Generator re-engages; loop paused.

Preview spawn fails

ComputeEnv adapter error after 3 retries.
Label infra-issue; on-call paged.

Out-of-scope request

Classifier ≥ 0.7 confidence.
Polite refusal + suggest /file-followup.

Reporter silent

7d / 14d timers.
Ping at 7d; abandon + destroy preview at 14d.

Reporter loop > 5

Iteration counter.
Label needs-human reason iter-cap.

Human loop > 3

Iteration counter.
Label needs-human reason human-iter-cap · sync review required.

Reviewer rubber-stamps

Hard to detect automatically.
CODEOWNERS checklist · pre-review reports from Security + Code agents · 10% sample audit.

Spend cap hit mid-PR

Cost tracker.
Pause · label needs-human reason budget-exceeded.

Adapter outage

e.g. Slack down.
Notifier falls back to email · LLM falls back via LiteLLM chain.

LLM hallucinates test results

Snapshot mismatch.
Gate 1 always re-runs in agentlab · agent's claim vs actual → escalation.

Prompt injection in reporter comment

Injection detector before classification.
Refuse · log to security audit queue · do not iterate.

Merge conflict with main

Repo adapter detect.
Auto-rebase 1×; on fail → label needs-human reason rebase-conflict.

10Interactions with the other 5 agents

📝

Spec Generator

Produces the input. Implementation Agent reads agent/spec-<N> HEAD; never wholesale-rewrites the spec. Spec Generator stays available for spec-refinement-needed requests.

🛠️

Code Agent

Posts an automated code-quality report on the PR before awaiting-human-review. Impl Agent surfaces top findings in the handoff comment to humans.

🛡️

Security Agent

Posts an automated security pre-review. Same surfacing rule. Tenancy-boundary issues block the human-handoff comment until acknowledged.

⚡

Optimization Agent

Doesn't directly interact. Its perf suggestions land as separate autonomous PRs through its own loop.

💬

Support Triage Agent

For chat-originating issues, Support Triage's bidirectional sync reads Impl Agent's PR labels to update the in-app chat thread when the preview is ready, when the PR merges, and when the wave-2 rollout completes.
